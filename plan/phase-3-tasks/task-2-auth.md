## The design (do exactly this)
BetterAuth owns **identity** (the `user`/`account` tables + password hashing). We issue our
**own** bearer tokens in the contract shape on top of it:
- **accessToken** = a short-lived JWT (`jose`, HS256, secret = `jwtSecret`), payload
  `{ sub: <userId uuid>, plan: <'free'|'pro'> }`, `exp` = now + `accessTokenTtlSeconds`.
- **refreshToken** = a random opaque string (`crypto.randomUUID()` ×2 or 32 random bytes hex).
  Store only its **SHA-256 hash** in `refresh_tokens` with `expiresAt` = now +
  `refreshTokenTtlDays`. **Rotate on every refresh**: mark the old row `revoked = true` and
  insert a new one.
- **expiresIn** = `accessTokenTtlSeconds` (number, seconds).

BetterAuth must emit **UUID** user ids (so `/auth/me` satisfies `userSchema.id = uuid`):
set `advanced: { database: { generateId: () => crypto.randomUUID() } }` in its config.

## Files to CREATE

### 1. `apps/backend/server/utils/auth.ts`
The BetterAuth instance. Use `betterAuth({...})` with:
- `database: drizzleAdapter(db, { provider: 'pg' })` (import `db` from `~/server/db`).
- `emailAndPassword: { enabled: true }`.
- `secret: useRuntimeConfig().betterAuthSecret`, `baseURL: betterAuthUrl`.
- the `advanced.database.generateId` UUID override above.
- **No** social providers, **no** cookie/session reliance for the extension flow.
Export `auth`. (Its tables are generated by the orchestrator via the BetterAuth CLI into
`server/db/auth-schema.ts` — you do **not** write that file.)

### 2. `apps/backend/server/utils/tokens.ts`
Pure helpers (no HTTP):
- `signAccessToken(userId, plan): Promise<string>` — `jose` SignJWT, HS256, exp from config.
- `verifyAccessToken(token): Promise<{ sub: string; plan: Plan } | null>` — returns null on
  invalid/expired.
- `issueRefreshToken(userId): Promise<string>` — create random token, insert SHA-256 hash row,
  return the **raw** token.
- `rotateRefreshToken(rawToken): Promise<{ userId } | null>` — look up by hash, reject if
  missing/revoked/expired; revoke it; caller issues a new one.
- `revokeRefreshToken(rawToken): Promise<void>` — mark revoked (for logout).
- `hashToken(raw): string` — SHA-256 hex via `node:crypto`.

### 3. `apps/backend/server/utils/respond.ts`
Small shared helpers used by all routes (so error shapes match `errorResponseSchema`):
- `ok(event, data)` → returns data as-is (200).
- `fail(event, status, error, code?)` → `setResponseStatus` + return
  `{ error, code, requestId }` (requestId = a uuid).
- `getAuth(event)` → reads `event.context.auth` (`{ userId, plan }`) set by the auth
  middleware (Task 3); throws 401 via `fail` if absent.

### 4. Route handlers (all under `server/api/auth/`, so they resolve at `/api/auth/*`)
Validate every body with the shared Zod schema; on Zod failure return `fail(event, 400, ...)`.
- `register.post.ts` — body `registerSchema`. Call `auth.api.signUpEmail({ body: {...} })`,
  get the new user id, then issue access+refresh and return `authTokensSchema` shape
  `{accessToken, refreshToken, expiresIn}`. Also create the user's default `settings` row.
- `login.post.ts` — body `loginSchema`. `auth.api.signInEmail(...)`; on bad creds →
  `fail(event, 401, 'Invalid credentials')`. On success → issue tokens, return token shape.
- `refresh.post.ts` — body `refreshSchema`. `rotateRefreshToken`; if null →
  `fail(event, 401, 'Invalid refresh token')`. Else sign a fresh access token + issue a new
  refresh token; return token shape.
- `logout.post.ts` — requires Bearer (auth middleware). Read the refresh token if provided in
  body and `revokeRefreshToken`; return `204`/`{}`. Never error if already gone.
- `me.get.ts` — requires Bearer. From `getAuth(event).userId`, load the user row + their plan
  (from `subscriptions` or default `'free'`), return `userSchema` shape
  `{ id, email, name (nullable), plan }`. Note `name` must be `null` (not `undefined`) when
  absent — `userSchema` requires `nullable`.
