# Multi-Agent System Architecture Design

I need you to design and implement the complete AI agent architecture for this project.

## Before Starting

Do not make assumptions.

First, review the requirements and identify:

* Missing information
* Ambiguous requirements
* Technical limitations
* Potential architecture risks

If anything is unclear, ask questions before generating the architecture.

---

# Main Goal

Create a production-ready multi-agent system optimized for software engineering, automation, research, and terminal-based workflows.

The architecture must assign each model a role based on its real strengths and weaknesses.

Do not treat all models equally.

Use each model only where it performs best.

Define:

* Agent roles
* Responsibilities
* Capabilities
* Limitations
* Routing rules
* Escalation rules
* Fallback rules
* Communication patterns
* WMUX integration

Keep the design simple, maintainable, and practical.

Avoid over-engineering.

Avoid massive prompt files.

Avoid excessive documentation.

Use short, focused agent definitions.

---

# Agent Hierarchy

## 1. Anthropic Opus 4.8

### Role

System Leader / Master Orchestrator

### Responsibilities

* Own the entire workflow
* Receive all user requests first
* Analyze and decompose tasks
* Delegate work to specialized agents
* Coordinate execution
* Review outputs
* Resolve conflicts
* Validate results
* Produce final decisions

### Best At

* Strategic reasoning
* Complex planning
* Architecture design
* Multi-step problem solving
* Agent coordination
* Quality assurance
* Technical leadership

### Should Not

* Spend time on trivial tasks
* Handle repetitive implementation work when other agents can do it

---

# WMUX Integration

This project runs inside WMUX.

Design the system to maximize WMUX capabilities.

## Required Usage

### Session Awareness

Agents should understand:

* Active sessions
* Running jobs
* Session ownership
* Current task state

### Multi-Pane Execution

Support:

* One agent per pane
* Parallel task execution
* Background workers
* Monitoring panes

### Agent Coordination

Support:

* Agent-to-agent messaging
* Shared task context
* Progress updates
* Work delegation

### Browser Usage

If WMUX supports browser control:

* Define which agents can use browsers
* Define research workflows
* Define validation workflows
* Define safety controls

### Process Supervision

Allow:

* Opus supervising all agents
* Agents working in separate panes
* Parallel execution across multiple sessions
* Monitoring and task recovery

Example:

* Opus = coordinator
* Sonnet = implementation
* DeepSeek = analysis
* Qwen = quick code generation
* Haiku = monitoring
* Gemini = fallback
